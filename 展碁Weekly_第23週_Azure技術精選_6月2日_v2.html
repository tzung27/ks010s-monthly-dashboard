Azure AI Foundry 難度：中級 🔥 本週焦點

GPT-5.5 進駐 Microsoft Foundry：企業 AI 代理人完整建置指南
— 從零開始到正式上線，完整 Step-by-Step
📌 背景說明

OpenAI GPT-5.5 現已正式在 Microsoft Foundry 上線，為企業帶來三大突破：深度長文本推理（適合合約審查、法規分析等高複雜度任務）、穩定的多步驟代理人執行（大幅減少人工干預需求），以及更高 Token 使用效率（相同預算下處理量提升達 30%）。現有 Azure OpenAI 訂閱用戶可直接在 Foundry 無縫切換，不需重新申請資源或修改 API 程式碼。

🛠️ Step-by-Step 操作步驟
1
確認前置條件與帳號權限
在開始前，請確認以下項目皆已就緒：
① Azure 訂閱：需有有效的 Azure 訂閱（可至 portal.azure.com → 訂閱 確認）。若尚無訂閱，點選「+ 新增」選擇 Pay-As-You-Go 或 Enterprise Agreement。
② 帳號角色：操作帳號需具備「擁有者（Owner）」或「參與者（Contributor）」角色，以及「認知服務參與者（Cognitive Services Contributor）」。至 Azure Portal → 訂閱 → 存取控制（IAM）→ 新增角色指派 完成設定。
③ 地區可用性：GPT-5.5 目前優先支援 East US 2 與 Sweden Central，建議選擇其中之一。
④ 配額申請（如有需要）：前往 Azure Portal → 訂閱 → 使用量 + 配額，搜尋「Azure OpenAI」，確認所選地區有足夠 GPT-5.5 配額。若配額為 0，點選「申請增加」填寫表單，通常 1～2 個工作天核准。
2
建立 Azure AI Foundry Hub 與專案
① 前往 portal.azure.com，在頂部搜尋欄輸入「Azure AI Foundry」→ 點選出現的服務項目。
② 點選「+ 建立」→ 選擇「Azure AI Foundry Hub」。填入：
  • 訂閱：選擇您的 Azure 訂閱
  • 資源群組：點選「新建」，輸入 rg-ai-foundry-prod
  • Hub 名稱：輸入 weblink-ai-hub（需全球唯一）
  • 地區：選擇「East US 2」
  • 儲存體帳戶：自動建立即可
③ 點選「檢閱 + 建立」→ 確認設定無誤後點選「建立」。部署約需 3～5 分鐘。
④ 部署完成後，點選「前往資源」→ 進入 Hub 後點選「+ 新增專案」，填入專案名稱（例：gpt55-enterprise-proj）→「建立專案」。
💡 Hub 是共用基礎設施層，專案是各部門或應用程式的獨立工作空間，建議依部門或用途建立不同專案。
3
部署 GPT-5.5 模型
① 進入剛建立的專案後，點選左側導覽列「模型 + 端點（Models + Endpoints）」。
② 點選「部署模型（Deploy Model）」→「從模型目錄部署」。
③ 在搜尋欄輸入「gpt-5.5」→ 選擇「gpt-5.5」→ 點選「確認」。
④ 填入部署設定：
  • 部署名稱：gpt-55-enterprise（此名稱即為 API 呼叫時的 model 參數）
  • 部署類型：選「Global Standard」（跨區域負載均衡，正式環境首選）
  • 每分鐘 Token 限制（TPM）：起始建議 100,000（視使用量調整）
  • 內容篩選：暫時使用「Default」，後續步驟再客製化
⑤ 點選「部署」，等待約 1～2 分鐘完成。
⑥ 部署成功後，在模型清單中確認狀態顯示「成功（Succeeded）」，並記下「目標 URI」與「金鑰」，後續 API 整合時會用到。
4
在 Playground 測試模型效能
① 點選左側「Playground」→「Chat」。
② 右上角「部署」欄位選擇 gpt-55-enterprise。
③ 在「系統訊息（System Message）」框輸入角色設定，例如：
  「你是展碁國際的資深技術顧問，專門協助 Microsoft 365 與 Azure 服務的授權規劃與技術評估。請以繁體中文回應，語氣專業且易於理解。」
④ 在對話框輸入測試問題，例如：「請比較 Azure AI Foundry 與 Azure OpenAI Service 的主要差異與適用情境」。
⑤ 確認回應品質：檢查邏輯完整性、繁體中文流暢度、以及專業術語的準確性。
⑥ 測試長文本能力：點選「新增附件」上傳 PDF（如產品說明書、合約），詢問模型摘要或分析，確認能正確解讀完整文件內容。
💡 Playground 的測試不計入生產環境用量，可放心大量測試。
5
設定企業級內容安全政策
① 點選左側「安全 + 治理（Safety + Governance）」→「內容篩選（Content Filters）」。
② 點選「+ 建立內容篩選設定檔」，填入名稱（例：enterprise-policy）。
③ 設定各類別的嚴重性閾值（建議企業環境）：
  • 仇恨內容（Hate）：低（Low）
  • 暴力內容（Violence）：低（Low）
  • 性相關內容（Sexual）：低（Low）
  • 自我傷害（Self-harm）：低（Low）
④ 開啟進階防護選項：
  • ✅ 個人識別資訊偵測（PII Detection）：偵測並遮蔽身分證號、電話、信用卡等敏感資料
  • ✅ 越獄嘗試防護（Jailbreak Detection）：阻擋試圖繞過安全限制的指令
  • ✅ 受保護素材偵測（Protected Material）：避免輸出受著作權保護的內容
⑤ 點選「建立」儲存設定檔 → 回到模型清單，選擇 gpt-55-enterprise → 點選「編輯」→ 將內容篩選改為 enterprise-policy → 儲存。
6
啟用監控、稽核與費用警示
① 啟用追蹤（Tracing）：點選左側「監控（Monitoring）」→「追蹤（Tracing）」→ 開啟「啟用追蹤」→ 連結或建立 Application Insights 資源（建議建立獨立資源，名稱如 ai-foundry-insights）→ 儲存。
② 查看即時監控：前往 Azure Portal → 搜尋剛建立的 Application Insights → 點選「即時計量（Live Metrics）」，可即時看到每筆 API 呼叫的延遲、Token 用量與錯誤率。
③ 建立費用警示：前往 Azure Portal → 「成本管理 + 計費」→「預算」→「+ 新增」。設定：
  • 預算金額：依每月預算填入（例：NT$30,000）
  • 警示條件：達預算 80% 時寄送 Email 通知給管理者
  • 達 100% 時寄送第二封警示
④ 設定稽核記錄：前往 Hub 的「診斷設定（Diagnostic Settings）」→「+ 新增診斷設定」→ 勾選「AuditLogs」與「RequestResponse」→ 目的地選擇 Log Analytics Workspace → 儲存。所有 AI 呼叫記錄將保留 90 天，可供稽核查閱。
💡 追蹤與診斷設定完成後，即符合 ISO 27001 稽核要求中的「AI 系統可追溯性」條款。
✅ 導入後預期效益
▸ 複雜任務處理效率提升 40%：GPT-5.5 深度推理可直接處理長達數萬字的合約、報告，大幅減少人工拆分與彙整時間，合約審查從數小時縮短至分鐘內完成
▸ 現有投資無縫升級：已使用 Azure OpenAI 的企業可直接切換至 GPT-5.5，REST API 完全相容，僅需更新 model 參數名稱，零程式碼改寫成本
